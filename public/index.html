📚 Como Funciona a Criptografia

🔑

Simétrica (AES)

  • Uma única chave secreta
  • Mesma chave cifra e decifra
  • Muito rápida
  • Problema: como compartilhar a chave?
AES-256-CBC
🔐

Assimétrica (RSA)

  • Par de chaves: pública + privada
  • Pública cifra, privada decifra
  • Mais segura para troca de dados
  • Mais lenta que a simétrica
RSA 2048 — Usada nesta API
⚡

Híbrida (RSA + AES)

  • RSA cifra a chave AES (32 bytes)
  • AES cifra os dados (sem limite)
  • Segurança do RSA + velocidade do AES
  • Padrão usado na prática (TLS/HTTPS)
Abordagem desta API

🧮 Fundamento Matemático do RSA

1
Gerar dois primos grandes p e q (cada um com ~1024 bits)
2
Calcular o módulo n = p × q
3
Calcular o totiente de Euler φ(n) = (p − 1) × (q − 1)
4
Escolher expoente público e = 65537 (padrão)
5
Calcular expoente privado d × e ≡ 1 (mod φ(n))
Cifrar c = me mod n
Decifrar m = cd mod n

🔄 Fluxo da Criptografia na API

👤 Cliente

1 GET /crypto/public-key
2 ← Chave pública RSA (PEM)
3 Gera chave AES → Cifra dados com AES → Cifra chave AES com RSA
4 POST /apostas (X-Encrypted: true) → { encryptedKey, iv, encryptedData }
5 Decifra chave AES com RSA privada → Decifra dados com AES → Processa
6 ← Resposta (201 Created)
